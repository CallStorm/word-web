# Web Mode Environment Variables

When `word-web` dispatches a job, these environment variables are injected into the Docker container.

| Variable | Required | Description |
|----------|----------|-------------|
| `JOB_ID` | Yes | Unique job identifier |
| `WORK_ROOT` | Yes | Project workspace, e.g. `/work/projects/<job_id>/` |
| `UPLOADS_DIR` | No | Upload staging dir, default `/work/uploads/<job_id>/` |
| `JOB_OPTIONS_JSON` | No | JSON blob with JobOptions fields |
| `PROMPT` | Yes | User prompt (also passed via claude `-p`) |

## JOB_OPTIONS_JSON Fields

```json
{
  "generation_mode": "freeform",
  "language": "zh | en",
  "scenario": "report | contract | letter | memo | academic | general",
  "tone": "formal | casual | technical | professional | concise",
  "audience": "general | executive | team | client | expert | student",
  "core_topic": "...",
  "outline": ["章节一", "章节二"],
  "key_points": ["要点一"],
  "generation_hint": "..."
}
```

Cover, TOC, tables, and figures are **not** configured here — decide from the user prompt only.

## Path Convention

```
/work/
├── uploads/<job_id>/
└── projects/<job_id>/              # WORK_ROOT
    ├── sources/summary.md
    ├── outline.md
    ├── .preview/page-1.png         # cover thumbnail (host-generated)
    └── exports/output.docx
```

Preview (cover PNG, HTML, outline) is generated by **word-web host** after finalize — the agent must **not** run preview scripts.
